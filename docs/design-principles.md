# 界面设计

本项目的界面不追求“花哨”，也不追求“后台味很重的工具感”。它更像一个高密度、低干扰、可快速扫读的学习工作台。

## UI / UX 的设计原因

### 信息优先

用户通常不是来欣赏页面，而是来判断：

- 今天有什么课
- 哪些作业和考试快到了
- 下一步该进入哪个对象

所以界面必须优先支持扫描、比较、切换，而不是先营造强装饰氛围。

### 轻量但不松散

页面应该保持克制：

- 依靠留白、边框、层级和少量强调色建立秩序
- 不依赖大面积强色块和厚重阴影
- 不让卡片、标签、按钮到处争抢注意力

### 交互要顺手，不要打断

这个项目的大部分操作都很短：

- 标记作业完成
- 改一个待办
- 订阅或退订班级
- 看一下评论

因此默认应优先就地完成。只有在破坏性确认或明显跨上下文时，才进入更重的交互层。

## 稳定的界面约束

- 首页和主要列表页应保持高信息密度，但仍可快速扫读
- 时间、状态、对象名应比说明文字更靠前
- 卡片和 tab 的职责必须清晰，不能只是视觉分组
- 链接负责导航，按钮负责动作，不要混用

## 常见交互判断

- 轻量编辑：优先 `sheet`
- 破坏性确认：使用 `dialog` / `alert-dialog`
- 需要持续消费信息：进入详情页
- 需要横向比较：使用表格
- 需要纵向扫描和逐条处理：使用列表或卡片
