# API 与 MCP

本项目对外能力分三层，各自解决不同问题。

## 站内能力

浏览器页面直接消费的能力。服务于首页判断、详情页聚合、列表页浏览和就地编辑。

## API 能力

REST / OpenAPI 提供清晰、稳定、可集成的资源面。回答的是外部客户端能围绕哪些对象集成、读写和鉴权边界是什么。

## MCP 能力

给工具型客户端（AI / agent）提供受约束的操作面。把工具调用与页面请求分开，权限和 scope 收敛到单独边界。

## 为什么三者同时存在

- 页面需要站内交互能力
- 第三方客户端需要稳定资源接口
- 工具调用需要更强约束的协议面

不应强行合并成一个入口。

## 改动指引

- 页面自身的数据拼装 → 站内页面与服务层
- 对外集成资源面 → `src/app/api`
- 工具型客户端接入 → `src/lib/mcp`
- 不要把页面临时需求扩散成公开 API
