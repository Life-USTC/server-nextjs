# E2E 覆盖度对照（页面功能级）

本文件用于回答两个问题：

- `src/app/**/page.tsx` 的页面路由是否都有对应 E2E：是（已全覆盖）。
- 每个页面里“用户能点的按钮/能提交的表单/能切换的 Tab”等关键功能是否都被测到：不完全（见下方缺口清单）。

## 总结

- 路由覆盖：页面路由与 API Route Handlers 现已基本都有对应 E2E；本轮进一步补齐了 `/?tab=calendar`、`/?tab=exams`、`/api/uploads*`，并让 `/admin/users` 分页契约与页面实现保持一致。
- 功能覆盖：关键身份流、待办 CRUD、OAuth 授权码流、上传 API 主路径、dashboard 主要 tab、多账户解绑主流程、以及内容页的 seed 过滤最短路径都已有可回归断言；剩余缺口主要在更深层的交互和破坏性分支。

## 关键缺口（建议优先补测）

按“用户高频 + 风险高 + 交互复杂”排序：

1. 内容页更深互动（`/courses`、`/teachers`、`/sections`）
   - 分页与多维筛选组合、以及更多真实控件交互仍缺
2. 班级/订阅的深层日历交互（`/?tab=subscriptions`、`/sections/[jwId]`）
   - 日历事件 hover/popover 详情与更多跳转分支仍未显式覆盖
3. 破坏性设置分支（`/settings/danger`）
   - 真实删除不测（出于测试环境破坏性考虑）
4. 上传相关的页面入口
   - 当前没有独立 dashboard uploads 页面；上传能力主要由 `/api/uploads*` 覆盖，若未来重新挂回 UI，需要补回页面层行为测试

## 说明

- 本覆盖对照是“页面功能级”静态对比：根据页面/关键组件里存在的按钮、表单、Tab、菜单等交互点与当前 E2E 断言/点击路径做差分。
- 对第三方渲染的页面（`/api-docs` Swagger UI），建议只做“关键渲染 + 关键交互一条最短路径”，避免不稳定。
